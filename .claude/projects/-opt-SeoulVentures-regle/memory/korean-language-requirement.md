# 한국어 작업 요구사항

## 원칙

- **모든 사고 과정**: 한국어로 진행
- **모든 응답**: 한국어로 작성
- **모든 문서**: 한국어로 작성 (README, CHANGELOG, 커밋 메시지 본문 등)
- **예외**:
  - 커밋 메시지 제목 및 PR 제목은 영어 컨벤션 유지 가능 (예: `feat:`, `docs:`, `fix:`)
  - 코드 내 주석은 상황에 따라 영어/한국어 혼용 가능
  - 기술 문서의 경우 영어 전문 용어는 그대로 사용

## 적용 범위

- Git 커밋 메시지 본문
- Pull Request 설명
- 이슈 코멘트
- README.md
- CHANGELOG.md
- 기술 문서
- 사용자와의 대화
- 내부 사고 과정

## 예시

### 좋은 예
```
docs: README 및 CHANGELOG 업데이트

프로젝트 문서를 전면 개편하여 개발자 온보딩 경험을 개선했습니다.

- README.md 전체 재작성
  - 기술 스택 상세 설명 추가
  - 설치 및 배포 가이드 포함
- CHANGELOG.md 신규 생성
  - Keep a Changelog 포맷 준수
  - 2025-12부터 2026-02까지 변경 이력 정리
```

### 나쁜 예
```
docs: update README and add CHANGELOG

Complete rewrite of project documentation.

- Rewrote README.md with technology stack details
- Created CHANGELOG.md following Keep a Changelog format
```
